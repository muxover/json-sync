# Json-Sync v0.1.0 — Initial release

**Release date:** 2026-03-01

### Install

```toml
[dependencies]
json-sync = "0.1.0"
shardmap = "0.1"
```

For the DashMap backend:

```toml
json-sync = { version = "0.1.0", features = ["dashmap"] }
dashmap = "6"
```

---

## Summary

First public release of Json-Sync — a thread-safe, persistent JSON-backed key-value store with pluggable map backends (ShardMap, RwLock<HashMap>, DashMap) and configurable flush policies (immediate, async, manual).

---

## What's included

### Core
- `JsonSync<K, V, M>` with `MapBackend` trait; implementations for ShardMap, `RwLock<HashMap>`, and DashMap (optional feature).
- Flush policies: `Immediate`, `Async(Duration)`, `Manual`.
- Crash-safe writes via temp-file-then-rename.
- Builder API with pretty-print JSON option.

### Operations
- `insert`, `get`, `remove`, `clear`, `update`, `get_or_insert`, `get_or_insert_with`, `extend`, `keys`, `values`, `iter`, `contains_key`, `len`, `is_empty`, `flush`, `path`.

### Quality
- `Debug` for `JsonSync`, `JsonSyncHandle`, `JsonSyncBuilder`; `#[must_use]` on read-only methods; `#[non_exhaustive]` on `Error` and `FlushPolicy`.
